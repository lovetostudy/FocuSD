# 待办去日期化方案（终版）

## 背景

问题：当前待办按日期分组，跨日自动归档。未完成任务被一并归档，用户需翻历史才能找到。用户要一个持久待办列表，不受日期影响。

## 新模型

```
待办列表（App 内）          磁盘文件（项目根目录）
┌─────────────────┐        ┌──────────────┐
│ ☐ 买咖啡         │   →    │  todos.md    │  ← 始终是当前未完成列表（覆盖写）
│ ☐ 写周报         │        └──────────────┘
│ ✓ 回复邮件 ──────┼──→    ┌──────────────────┐
│ ✓ 提交代码 ──────┼──→    │  2026-07-11.md   │  ← 今天完成的（追加写）
└─────────────────┘        ├──────────────────┤
                           │  2026-07-10.md   │  ← 昨天完成的
                           └──────────────────┘
```

- 未完成项 → 列表中永远保留，覆盖写 `todos.md`
- 标记完成 → 从列表移除，追加写入当天 `YYYY-MM-DD.md`（格式 `- [x] 任务标题`）
- 取消完成 → 恢复到列表 + 重写 `todos.md`，日期文件里保留不删
- 没有日期切换/自动归档逻辑

## 页面结构（待办页 Spine 三个点）

| 点 | 名称 | 行为 |
|----|------|------|
| 1 | 待办列表 | 未完成任务列表 + 底部「已完成」折叠区（默认折叠，点击展开看已完成的） |
| 2 | 保存 | 覆盖写 `todos.md`（或改自动保存，去掉手动触发） |
| 3 | 完成日记书 | 扫描项目根目录 `YYYY-MM-DD.md`，解析已完成项，按日期分组展示 |

## 完成日记书（点3）

**数据来源**：直接读磁盘 `YYYY-MM-DD.md` 文件，不经过 localStorage。

**新增 Rust 命令**：
```rust
#[tauri::command]
fn list_completed_archives(directory: String) -> Result<Vec<CompletedArchive>, String>
```
扫描目录下 `YYYY-MM-DD.md`，解析每行的 `- [x] 任务标题`，按日期分组返回。

**前端展示**：保留旧的卡片视图和时间线视图两种布局，UI 复用现有 `ArchiveBrowser` 组件，但数据来源从 `localStorage` 改为 Rust 命令返回值。

**localStorage 同步**：不需要。磁盘是唯一数据源。

## 删除内容

### 前端（`src/App.tsx`）
- 日期切换逻辑：`currentTodoDate`、`loadCurrentTodoDate`、`rolloverToToday`、`TODO_DATE_STORAGE_KEY`、`TODO_LAST_SAVED_SIGNATURE_STORAGE_KEY`
- 每日笔记：`dailyNote` 状态、`DAILY_NOTE_STORAGE_KEY`、笔记编辑 UI
- 类型简化：删除 `TodoPageMode` 中的 `"archive"` `"review"`（保留列表/日记书两个独立状态）、删除 `ArchiveLayout` 用独立枚举替代
- `upsertArchive` 函数
- `localStorage` 旧 key 不再读写：`focusd-island-current-date`、`focusd-island-archives`、`focusd-island-last-saved-signature`、`focusd-island-daily-note`

### 样式（`src/App.css`）
- 每日笔记编辑相关样式
- Todo 页面 Tasks/DAILY tab 切换样式

### Rust 后端（`src-tauri/src/lib.rs`）
- 删除 `save_todo_markdown` 命令
- 新增三个命令：
  - `save_todos(file_path, content)` — 覆盖写 `todos.md`
  - `append_completed_todo(file_path, line)` — 追加写 `YYYY-MM-DD.md`
  - `list_completed_archives(directory)` — 扫描 `YYYY-MM-DD.md` 返回已完成归档列表

## 保留 & 改动

| 内容 | 改动 |
|------|------|
| 待办 CRUD | 完成时追加写磁盘、从列表移除 |
| 专注任务 | 不动 |
| 未完成计数 | 不动 |
| Spine 三个点 | 点1 去掉 tab 只显示列表；点2 行为改为保存 `todos.md`；点3 数据源改为磁盘 |
| ArchiveBrowser 组件 | 保留 UI（卡片+时间线），数据改从 Rust 命令读 |
| 设置页 | 去掉笔记保存目录选择器，改用固定目录（项目根目录） |

## 新的存储文件结构（项目根目录）

```
FocuSD/
├── todos.md           ← 未完成列表（覆盖写）
├── 2026-07-11.md      ← 完成归档（追加写，按日期）
├── 2026-07-10.md
└── ...
```

## 数据迁移

旧 `localStorage` 中的归档数据直接丢弃。已完成归档直接读新的磁盘文件。

## 影响范围

| 层面 | 变化量 |
|------|--------|
| `src/App.tsx` | 删除约 300+ 行（日期逻辑、笔记），新增约 30 行（磁盘读取回调），改造完成逻辑 |
| `src/App.css` | 删除 daily-note 样式，保留 archive 样式 |
| `src-tauri/src/lib.rs` | 删除 1 个旧命令，新增 3 个新命令 |

## 验证

1. `pnpm build` 通过
2. `pnpm tauri dev` 手动验证：
   - 添加、编辑、完成、删除待办
   - 已完成项出现在列表底部折叠区 + 追加写入 `YYYY-MM-DD.md`
   - 点3「完成日记书」显示所有日期文件，卡片/时间线切换正常
   - 手动改系统日期，待办不受影响
   - 取消完成 → 恢复到列表，日期文件里记录保留
